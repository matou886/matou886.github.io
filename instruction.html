Usermanu

Prepare for using

To use the CheckMaster, user have to configure both the front-end and the back-end. For the front-end part, we present the user interface to user, allowing user to perform actions regarding with checklists. The back-end part is to provide the server, the database to the front-end, making the front-end operate properly. Once the configurations are all set, user just need to run the back-end and launch the front-end to get the CheckMaster working.

Front-End

Note: this application is only available for Windows users in this version

If you simply want to get the executable software, please refer to Installation. If you want to work on the project, you need to configure the workspace.

Installation

To install the application, you can simply find the executable setup file under the folder: checkmaster-front-end\dist_electron. You can get checkmaster-front-end Setup 0.1.0.exe there, which will set up our software for you. After setting up, you can just double click the software checkmaster-front-end to launch our program.

Environment configuration

  • Node.js

    Node.js download

    using

    node -v
npm -v

    to check the version.

  • Vue

    install Vue-CLI by using the command:

    npm install -g @vue/cli
# OR
yarn global add @vue/cli

    using

    vue --version

    to check

  • Electron Add electron to Vue

    vue add electron-builder

  • Run the project Open terminal under folder checkmaster-front-end\checkmaster-front-end, run:

    npm run electron:serve

Back-End

  1. Download and unzip the compressed package.
  2. Download MongoDB, MongoDB Tools and MongoDB Campass, install MongoDB and MongoDB Tools in the same path. (e.g., C:\Program Files\MongoDB).
  3. Open MongoDB Campass, click Connect to create a new database, and create a new database named it checkmaster.
  4. Open the command prompt as administrator, enter the path of the MongoDB Tools, then run following commands:
mongoimport -d checkmaster -c nodes [node.dat]
mongoimport -d checkmaster -c projects [project.dat]
mongoimport -d checkmaster -c users [user.dat]
[node.dat], [project.dat], [user.dat] need to be replaced by the path of these three files in your unziped compressed package

Now you should see three collections in your checkmaster database. 5. Open the checkmaster-server project with IntelliJ IDEA, make sure you have Java enviroment and maven enviroment. Then in the maven console, reload all maven projects, then unfold the lifecycle, ckick compile to compile the project. 6. After successfully build the project, you can run ServerApplication. Before running, make sure that port number 8080 is available in your device. 7. Since the server is running, we can using APIs to access database. First make sure your device has connected to WLAN. Open the command prompt, run following command ipconfig. Then find your IPv4 address of WLAN and you can access the following url in the browser to check if the server and database running successfully:

http://[Your IPv4 address]:8080/user/get?email=scyqw5@163.com

It returns the information data of the user scyqw5@163.com as JSON format. 8. To connect the front end to the database, open request.js in the utils file, then replace the value of the variable const baseurl with http://[Your IPv4 address]:8080. 9. Finally, with the WLAN connection and the server project keep running, the software can use APIs and access database successfully.
